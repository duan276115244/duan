# 02 - 核心架构

本章覆盖项目的入口文件、Bootstrap 装配流程、EnhancedAgentLoop 主循环及状态机设计。

## 入口文件

### 1. `src/entry.ts` — 统一 CLI 入口

所有 CLI 模式的分发器，根据 `args[0]` 分流到不同启动路径。

**核心函数**：
- `main()` (`entry.ts:89`) — 命令分发主函数
- `startAgent()` (`entry.ts:35`) — 动态 import `duan-v19.0.js` 并调用 `runDuan()`

**启动调用链**：
```
main()
 ├─ args.includes('--new-window') → launchNewWindow()
 ├─ args[0] === 'web'   → import('./web-server.js').start()
 ├─ args[0] === 'setup' → runSetupWizard() → startAgent()
 ├─ args[0] === 'pairing' → handlePairingCommand()
 ├─ args[0] === 'profile' → handleProfileCommand()
 ├─ args[0] === 'gateway' → handleGatewayCommand()
 └─ 默认路径：
     ├─ 首次运行检查（.setup-done 标记）
     ├─ config.applyEnv()
     ├─ discoverFreeModels()（后台探测 Ollama）
     ├─ 若 WEB_PORT 存在 → import('./web-server.js').start()
     └─ await startAgent()  ← 进入 TUI 主循环
```

**关键设计**：
- `entry.ts:184-201` 启动时自动探测免费模型（Ollama 等），失败不阻塞
- `entry.ts:208-218` 后台并行启动 Web 服务器（fire-and-forget）

### 2. `src/web-server.ts` — Web 服务器入口

Express HTTP 服务，默认端口 3001，端口冲突自动递增（最多 10 次）。

**启动流程**：
1. `setupMiddleware(app)` (`web-server.ts:39`) — 注册中间件
2. `loadConfig()` → `setAppConfig()` — 加载 `~/.duan/config.json`
3. `setupAgentLoop()` (`web-server.ts:75`) — **关键**：调用 bootstrap 装配核心模块
4. `setToolRegistry(registry)` (`web-server.ts:78`) — 注入到 `tools.ts`
5. `heartbeat.start()` (`web-server.ts:120`) — 启动心跳（每 10 分钟触发自进化）
6. `CapabilityAssessor` 初始化 — 加载 10 维度能力评估
7. `registerRoutes(app, serverCtx)` (`web-server.ts:237`) — 注册所有 REST/SSE 路由
8. `getMCPSecurityGuard().enablePollingMode()` — 启用 MCP 审批轮询
9. `setupRemoteBridge(app, port)` — 初始化远程交互桥接（飞书/Telegram/微信）

**ServerContext 注入**（`web-server.ts:175-236`）：
- `serverCtx` 对象包含 40+ 字段：`modelLibrary`、`conversations`、`agents`、`systemPrompt`、`loop`（统一主循环）、`capabilityAssessor`、`heartbeat`、`selfEvolve` 等
- `loop: unifiedLoop` — 将 `EnhancedAgentLoop` 实例暴露给 `chat-routes`，使 Web 请求默认走统一主循环

### 3. `src/duan-v19.0.ts` — TUI 主循环

**核心函数**：`export async function runDuan(): Promise<void>` (`duan-v19.0.ts:150`)

**REPL 循环**：
```
runDuan()
 ├─ new ConfigManager() + applyEnv()
 ├─ printLogo() + printWelcome()
 ├─ 首次配置检查（inquirer 引导）
 ├─ setupAgentLoop(callbacks)  ← 带审批回调
 ├─ setToolRegistry(registry)
 ├─ loadHistory() / saveHistory()
 └─ while (true) REPL:
      ├─ inquirer.prompt 等待输入
      ├─ 内置命令处理（exit/clear/help/config/switch）
      ├─ /image <path> [prompt]  多模态图片输入
      ├─ handleCommand(input)  34 个 CLI 子命令
      └─ processWithAgent(input)  调用 loop.run() 流式消费
```

**processWithAgent 核心逻辑** (`duan-v19.0.ts:330-453`)：
- `enhancedAgentLoop.run(input, history, undefined, { images })` — 异步生成器
- `for await (const event of agentLoop)` — 消费 7 类事件：`text`、`tool_call`、`tool_result`、`think`、`error`、`warning`
- `startSpin()/stopSpin()` — Braille 动画思考指示器

### 4. `desktop/main.js` — Electron 桌面入口

约 6600 行，**不直接运行 Agent**，而是通过 `spawn('node', 'dist/entry.js')` 派生子进程。

**生命周期**：
```
app.requestSingleInstanceLock()
 ├─ app.whenReady().then(() => {
 │    ├─ setupFileLogging()
 │    ├─ CSP 策略注入
 │    ├─ registerIpcHandlers()  ← 注册所有 IPC 处理器
 │    ├─ createMainWindow()     ← 加载 frontend/dist
 │    ├─ createTray()
 │    ├─ startAgent()           ← spawn 子进程
 │    ├─ startConfigWatcher()
 │    └─ setupAutoUpdater()
 │})
 ├─ app.on('window-all-closed') → 不退出，保持托盘运行
 └─ app.on('before-quit') → stopAgent() + destroyTray()
```

### 依赖注入机制

1. **构造器注入** — 模块构造时显式传入依赖
2. **`toolContext` 全局注入** — 150+ 模块通过 `toolContext.xxx = module` 注入到工具上下文（23+ 字段）
3. **`injectXxx` 方法链** — `createAgentLoop` 中对 `EnhancedAgentLoop` 调用 30+ 个 `injectXxx` 方法：
   ```ts
   loop.injectModelLibrary(modules.modelLibrary, modules.selfLearningSystem);
   loop.injectStrategyEngine(modules.strategyEngine);
   loop.injectSessionPersistence(modules.sessionPersistence);
   loop.injectModelRouter(modules.modelRouter);
   loop.injectAdversarialVerifier(modules.adversarialVerifier);
   loop.injectDuanPersonaEngine(personaEngine);
   // ... 25+ 个
   ```
4. **`setAgentRunner` 注入** — `loop.run.bind(loop)` 注入到 `SubAgentOrchestrator`、`BackgroundAgentManager`
5. **`standardToolModules` 批量工具注册** — 51 个模块通过 `getToolDefinitions(): ToolDef[]` 契约统一注册

### 资源清理

- `cleanupFns: Array<() => void>` 收集所有定时器/监听器清理函数
- `pendingCleanupTimers` 暂存区解决"cleanupFns 声明前创建的定时器"问题
- `dispose()` / `disposeAsync()` 统一释放
- 关键：`identityNetwork.persist()` + `kgMemory.persist()` 在 dispose 前异步持久化

## EnhancedAgentLoop 主循环

### `src/core/enhanced-agent-loop.ts` — 核心（3900+ 行）

**`run()` 方法签名**（L2274-2279）：
```typescript
async *run(
  input: string,
  context: Array<{ role: string; content: string }>,
  customSystemPrompt?: string,
  options?: { images?: Array<{ url: string; mimeType?: string }> },
): AsyncGenerator<LoopEvent, TerminalReason, void>
```

### 状态机（AgentFSM）

基于 `src/core/agent-fsm.ts`，7 种状态：

```
IDLE → THINKING → EXECUTING → WAITING_HUMAN → COMPLETED
                                ↓
                              PAUSED
                                ↓
                              ERROR
```

合法转换：
- `IDLE → THINKING`
- `THINKING → EXECUTING / COMPLETED / ERROR`
- `EXECUTING → WAITING_HUMAN / THINKING / ERROR`
- `WAITING_HUMAN → EXECUTING / THINKING`
- `PAUSED → THINKING / EXECUTING / WAITING_HUMAN`（恢复）
- 全局可转：`PAUSED / ERROR / COMPLETED`

### 四阶段循环（Plan / Execute / Reflect / Learn）

#### Phase 0: 预检与初始化（L2286-2399）
- 检查 API Key，未配置直接 yield error
- **API 连通性预检**（5min 缓存）：流式请求 `max_tokens=1` 读首 chunk 即 abort
- `new AgentFSM()` + `transition(THINKING)`
- 生命周期钩子 `ON_SESSION_START`
- 四级记忆 `memoryStore.syncFromScratchpad`
- 初始化 `TaskPlanner` + `ResultReflector`
- 加载跨会话历史经验到 `lessonsLearned`

#### Phase 1: Planning（L2443-2518）
- **经验包匹配** — `ExperiencePackSystem.getReusableExperience(input)` 命中则直接注入执行路径参考
- **最优路径选择** — `OptimalPathSelector.selectOptimalPath`
- **推理引擎接入** — `reasoningEngine.think(input, context)` 使用 CoT/ToT/ReAct
- `plan = await this.planner.plan(input, toolNames)` — 调用 LLM 生成结构化 `ExecutionPlan`
- **计划审查回调** — 非简单任务让用户审批（approve/modify/reject）

#### Phase 2: Execute（主 while 循环，L2952-3900+）
```
while (true) {
  state.turnCount++;
  // 1. FSM 暂停检查（5min 超时自动终止）
  // 2. FSM 转入 THINKING
  // 3. 认知引擎决策（NN 分析上下文输出策略建议）
  // 4. SOP 自动匹配（历史成功经验直接注入）
  // 5. Scratchpad 事实提取
  // 6. 轮次限制检查（自适应扩展，剩余 ≤3 步时 +20 轮，只允许一次）
  // 7. 三级上下文阈值保护（90% 警告 / 98% 阻断）
  // 8. 上下文压缩（shouldCompact → compactMessages）
  // 9. 死循环检测（DOOM_LOOP_THRESHOLD=3）
  // 10. 收益递减检测（MAX_DIMINISHING_RETURNS=5 轮无进展自动结束）
  // 11. 计划重评估（每 15 轮重新规划）
  // 12. LLM 流式调用
  // 13. 工具调用执行（executeToolCallsWithApproval）
}
```

**工具执行核心**：`executeToolCallsWithApproval` (L4865)
- 调用 `ApprovalGate` 进行风险分类（safe/moderate/dangerous/critical）
- safe/moderate 自动通过，dangerous/critical 触发 `approvalCallback`

#### Phase 3: Reflect（任务完成后，L3714-3798）
- **反思引擎提取 SOP** — `extractSOP`，产出标准化操作流程
- **VFS 同步** — SOP 写入 `viking://skills/{name}/v{version}`
- **SkillRegistry 注册** — `registerFromSOP(sop)`
- **经验包自动提取** — `ExperiencePackSystem.autoExtractFromExecution`（零 token）
- **四级记忆整合** — `memoryStore.consolidate()`
- 生命周期钩子 `ON_SESSION_END`

#### Phase 4: Learn（贯穿全程的反馈闭环）
- `ToolLearningSystem.record` — 每次工具调用记录
- **失败教训提取** — 连续失败 ≥2 次生成 lesson 并持久化
- **强化学习闭环** — `smartToolSelector.markSuccess/markFailed` + `injectToolWeights`
- **EvolutionMetrics 喂养** — `_recordOutcomeToEvolutionMetrics` 是唯一反馈汇聚点

### LoopEvent 类型

```typescript
// agent-loop-types.ts
type LoopEvent =
  | { type: 'text'; content: string }
  | { type: 'tool_call'; toolName: string; toolArgs: string }
  | { type: 'tool_result'; toolName: string }
  | { type: 'think'; content: string }
  | { type: 'plan'; plan: ExecutionPlan }
  | { type: 'proactive_announcement'; severity: string }
  | { type: 'error' | 'warning' | 'compact' | 'done' | 'state' }

type TerminalReason =
  | { type: 'completed'; summary: string }
  | { type: 'max_tokens'; summary: string }
  | { type: 'error'; error: string; recoverable: boolean }
  | { type: 'interrupted'; summary: string }
  | { type: 'doom_loop'; toolName: string; summary: string }
  | { type: 'compaction_failure'; summary: string }
```

## 入口文件关系

```
                    ┌─────────────────┐
                    │  duan.bat /     │
                    │  npm run duan   │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
       npm run duan    npm run duan:web  electron .
              │              │              │
              ▼              ▼              ▼
        ┌──────────┐  ┌──────────┐  ┌──────────────┐
        │entry.ts  │  │entry.ts  │  │desktop/      │
        │(默认)    │  │(web 子命令)│  │main.js       │
        └────┬─────┘  └────┬─────┘  └──────┬───────┘
             │             │               │ spawn
             ▼             ▼               ▼
   ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
   │duan-v19.0.ts │ │web-server.ts │ │entry.ts      │
   │(TUI REPL)    │ │(Express HTTP)│ │(子进程)      │
   └──────┬───────┘ └──────┬───────┘ └──────┬───────┘
          │                │                │
          └────────────────┼────────────────┘
                           ▼
              ┌────────────────────────┐
              │  bootstrap.ts          │
              │  setupAgentLoop()      │
              │  ├─ createCoreModules()│
              │  ├─ createToolRegistry()│
              │  └─ createAgentLoop()  │
              └───────────┬────────────┘
                          ▼
              ┌────────────────────────┐
              │  enhanced-agent-loop.ts│
              │  loop.run()            │
              │  Plan→Execute→Reflect │
              │  →Learn               │
              └────────────────────────┘
```

**关键关系**：
1. `entry.ts` 是统一分发器
2. `web-server.ts` 与 `duan-v19.0.ts` 共享 `setupAgentLoop()`，区别仅在审批回调
3. `desktop/main.js` 不直接运行 Agent，派生子进程运行 `entry.ts`
4. `bootstrap.ts` 是唯一装配点，避免双实例
5. `enhanced-agent-loop.ts` 是唯一执行核心，通过 `loop.run()` 异步生成器统一被三种入口消费
