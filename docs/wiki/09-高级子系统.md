### 1.3 双神经网络模拟

项目通过两个神经网络实现**功能等价的双网络架构**（对应神经科学的 DMN + TPN）。位于 [consciousness-system.ts](file:///d:/good/jws/src/core/consciousness-system.ts) L230-251：

### 2.4 ExtendedThinkingService Phase D1 流式契约

**文件**：[extended-thinking-service.ts](file:///d:/good/jws/src/core/extended-thinking-service.ts)
**ThinkingPhaseEvent 接口**（L29-36）：
```typescript
export interface ThinkingPhaseEvent {
  emoji: string;  // 阶段标识 emoji
  title: string;  // 阶段标题
  body: string;   // 阶段内容
}
```

**runExtendedThinkingStream 异步生成器**（L75-138）严格遵循 **6 阶段 emoji 契约**：

| Emoji | 阶段 | 实现方法 | 跳过条件 |
|---|---|---|---|
| 🧩 | 问题分解 | `decomposeProblem` | 总是执行 |
| 🎯 | 约束识别 | `identifyConstraints` | 总是执行 |
| 💡 | 方案生成 | `generateSolutions` | shallow 模式跳过 |
| 🔍 | 边缘情况枚举 | `enumerateEdgeCases` | shallow 模式跳过 |
| ⚠️ | 风险评估 | 内联实现 | 仅 deep 模式执行 |
| 📚 | 相关经验检索 | 需 memoryOrchestrator | 无 MO 时跳过 |

**契约关键点**：
- 这些阶段函数均为**纯规则函数**，不调用 LLM，不阻塞首 text chunk
- `⚠️` emoji 有**双重用途陷阱**：既是风险评估阶段标记，又是运行时告警前缀（如"⚠️ 模型 X 使用非流式模式"）。前端 `ThinkingTrace.tsx` 必须用严格标题白名单正则 `^(emoji)\s+(known titles)$` 区分，不能仅按 emoji 前缀分类。
- `runExtendedThinking` 输出格式从 `## 标题` markdown 改为 `emoji 标题\n body` 后，旧测试 `expect(result).toContain('问题分解')` 仍通过（子串匹配），无需改测试。

## 关键架构总结

1. **意识状态命名差异**：实际实现是 `awake/focused/creative/reflective/dreaming` 5 状态（非 WAKING/DREAMING/DEEP_SLEEP/MEDITATION/FLOW）。`cognitive-state.ts` 另有 7 种 Mood + 4 级 ConsciousnessLevel，是补充的状态模型。

2. **双神经网络命名**：项目通过 `decisionNetwork`（对应 TPN 任务正向）+ `emotionNetwork`（对应 DMN 内省自我参照）实现**功能等价**，但未严格按神经科学命名。

3. **AES-256-GCM 双重用途**：
   - **运行时**：API Key 加密存储（`security-middleware.ts` + `unified-config.ts`）
   - **安装包**：源代码加密（`_decrypt.cjs`，不在 src/core 中）

4. **SelfQuestioningEngine 非独立文件**：该能力内嵌在 `self-evolution-engine.ts` 的 `selfQuestioning()` 方法中。

5. **competitive-comparison 实际名称**：实际文件名为 `continuous-evolution-system.ts`，包含完整的竞品爬取/分析/对比/路线图闭环。

6. **cleanupFns 契约严格遵守**：`CognitiveOrchestrator` 严格遵循项目记忆契约：3 处 `EventBus.on` 调用全部捕获 unsubscribe 句柄到 `cleanupFns`，`shutdown()` 中统一释放。

7. **Checkpoint 防损坏设计**：`checkpoint-rewind.ts` 的 3 步原子写入（tmp → hash 验证 → rename）直接对应项目记忆中"自动化 checkpoint 会损坏 .ts 文件"的教训，从设计层面杜绝写入中断导致的文件损坏。

8. **MCP 三层安全防护**：插件信任审核 + 工具调用审批门控 + 执行沙盒隔离，与系统级 4 级沙箱（none/vm/process/docker）形成纵深防御。
