## 3. 安装与启动流程

### 3.1 安装流程

**标准安装**：`npm install`
- `postinstall` 钩子（L44）：仅 `echo` 提示信息，无依赖编译或二进制下载。puppeteer/electron 的原生安装由各自包的 postinstall 处理。

**Windows 一键安装**：[install.bat](file:///d:/good/jws/install.bat)
1. 检测 Node.js（`where node`），未安装则提示去 https://nodejs.org/ 下载 v18+
2. 打印 Node 版本
3. 执行 `call npm install`（首次约 2-5 分钟）
4. 失败时建议切换淘宝镜像：`npm config set registry https://registry.npmmirror.com`

## 4. 构建流程

### 4.1 后端构建脚本

文件：[scripts/build.cjs](file:///d:/good/jws/scripts/build.cjs)（38 行，CommonJS）

执行步骤：
1. **TypeScript 编译**（L7-12）：`npx tsc`，**注意**：即使有类型错误也会输出 JS（catch 块仅打印警告，不退出）
2. **复制模板**（L14-20）：`fs.cpSync('templates', 'dist/templates', { recursive: true })`
3. **创建 duan.cmd**（L22-30）：写入 `@echo off\r\nnode "%~dp0dist\\entry.js" %*`，写入前先 `attrib -R` 清除只读属性（P5 修复：上次构建设了只读会导致 writeFileSync EPERM）
4. **设置只读**（L33-36）：`attrib +R duan.cmd` 防止误删

### 4.2 TypeScript 编译配置

文件：[tsconfig.json](file:///d:/good/jws/tsconfig.json)

- `target`: `ES2022`，`module`/`moduleResolution`: `NodeNext`（ESM）
- `outDir`: `./dist`，`rootDir`: `./src`
- **`strict`: false**，`noImplicitAny`: false（非严格模式，逐步治理 any）
- `esModuleInterop`: true，`skipLibCheck`: true
- 生成 declaration + declarationMap + sourceMap
- `include`: `["src"]`
- `exclude`（L20-28）：`node_modules`、`dist`，以及多个历史遗留文件（`src/index.ts`、`src/jarvis*.ts`、`src/duan.ts`、`src/core/agent.ts`、`src/agents/multi-agent.ts`、`src/models/*.ts`、**`src/core/__tests__/**/*.ts`**）

**重要**：测试文件被 tsconfig 排除，因此 ESLint 配置对测试文件禁用了类型信息（见 §6）。

### 4.3 前端构建

文件：[frontend/vite.config.ts](file:///d:/good/jws/frontend/vite.config.ts)

- 插件：`@vitejs/plugin-react` + `@tailwindcss/vite`（Tailwind v4）

### 4.4 Electron 打包配置

electron-builder 配置位于根 [package.json](file:///d:/good/jws/package.json) 的 `build` 字段（L107-173）。

## 10. CI/CD

### 10.1 GitHub Actions 工作流

文件：[.github/workflows/ci.yml](file:///d:/good/jws/.github/workflows/ci.yml)（单文件，名为 `CI`）

**触发条件**：
- push 到 `main`/`master`
- PR 到 `main`/`master`

**Job**：`verify`，runs-on `windows-latest`，timeout 30 分钟

**步骤**（L30-75）：
1. `actions/checkout@v4`
2. `actions/setup-node@v4`（Node 20，npm cache）
3. `npm ci`（严格安装）
4. `npm run build:frontend`（前端构建——因为后端 typecheck 不覆盖 frontend）
5. `npm run typecheck`
6. `npm run lint`
7. `npm run test`
8. **Capability baseline**（首次建立）：检查 `%USERPROFILE%\.duan\capability-assessment\baseline.json`，不存在则 `npm run capability:baseline`
9. **Capability regression check**：`npm run capability:check`（总分下降 ≥10 分或维度分下降 ≥15 分则失败）
10. **Upload artifact**：上传 capability 报告（`always()` 即使失败也上传）

**已知问题**：步骤 8、9 引用的 `npm run capability:baseline` / `capability:check` **未在 `package.json` scripts 中定义**。这会导致 CI 在该步骤报 `npm ERR! Missing script`。需补充这两个脚本：
```json
"capability:baseline": "tsx src/core/capability-assessment/cli.ts baseline",
"capability:check": "tsx src/core/capability-assessment/cli.ts check"
```

### 10.2 CI 与本地一致性

- CI 跑 `typecheck + lint + test` 三连，对应本地 `npm run verify:all`
- CI 额外跑前端构建与能力评估回归（本地 verify:all 不含这两项）
- CI 使用 Windows runner（与项目目标平台一致，Desktop 是 Windows 优先）
