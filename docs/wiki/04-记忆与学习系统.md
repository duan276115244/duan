## 记忆系统

### 双轨分层模型

项目存在**两套并行的分层模型**：

#### Hermes 三级架构（`hierarchical-memory.ts`）

```typescript
export enum HermesTier {
  L0_SESSION = 'L0_SESSION',         // 会话级（短期）
  L1_PERSISTENT = 'L1_PERSISTENT',   // 持久级（中期，90 天 TTL）
  L2_SKILL = 'L2_SKILL',             // 技能级（长期，不衰减）
}
```

**层级提升逻辑**（`memory-orchestrator.ts:1377` `inferTier()`）：
- `best_practice` / `pattern` / `achievement` → **L2 技能级**
- `user_preference` / `preference` / `fact` → **L1 持久级**
- `importance >= 7` → **L1 持久级**
- 默认 → **L0 会话级**

#### UnifiedMemoryManager 四层架构（`memory/manager.ts:35`）

```typescript
const HALF_LIVES = {
  working:    60 * 60 * 1000,              // 1 小时
  short_term: 6 * 60 * 60 * 1000,          // 6 小时
  long_term:  7 * 24 * 60 * 60 * 1000,     // 7 天
  core:       Infinity,                     // 永不衰减
};
```

提升链路：`working → short_term → long_term → core`（`promote()` L264）

自动提升规则（`enforceLimit()` L500）：`accessCount > 5 且 importance > 0.7` 的短期记忆自动提升为长期。

### 四种记忆类型

`hierarchical-memory.ts` 同时实现四种认知科学记忆类型：
- **WorkingMemory（工作记忆）**：1 小时半衰期，`WORKING_COMPRESS_THRESHOLD = 15`，超阈值触发 `autoCompressWorkingMemory()`
- **EpisodicMemory（情景记忆）**：任务事件序列
- **SemanticMemory（语义记忆）**：事实知识
- **ProceduralMemory（程序记忆）**：技能 / 操作流程

### 四级降级检索策略

`unified-memory.ts` 实现：

```
FTS5 全文检索（BM25 评分）  →  向量语义检索（余弦相似度）
        ↓ 失败                       ↓ 失败
   标签匹配检索              →    关键词模糊检索
```

- **FTS5 索引**：`FTSIndex` 类，Trigram 倒排索引 + BM25 评分 + LRU 查询缓存，检索延迟 < 10ms
- **向量检索**：`VectorStore.search(query, topK=5)`，关键词预过滤 + 余弦相似度
- **EmbeddingProvider**：`CompositeEmbeddingProvider`，OpenAI 1536 维主 + TF-IDF 512 维降级

### 衰减机制

**指数衰减 + 访问频率减缓**（`memory-orchestrator.ts:1062-1089`）：

```typescript
applyImportanceDecay(): number {
  const ageMs = Date.now() - entry.timestamp;
  const accessBonus = Math.min(entry.accessCount * 0.1, 0.5);
  const adjustedHalflife = DECAY_HALFLIFE_MS * (1 + accessBonus * 0.5);
  const decayFactor = Math.exp(-Math.LN2 * ageMs / adjustedHalflife);
  entry.importance *= decayFactor;
}
```

关键参数：
- `L1_TTL_MS = 90 天`（L1 持久级过期清理）
- `DECAY_HALFLIFE_MS = 30 天`（重要性衰减半衰期）
- 访问频率减缓衰减：访问越多衰减越慢

### 上下文管理

**ContextManager**（`context-manager.ts:179`）：
- 角色权重：system 1.0 / user 0.9 / assistant 0.6 / tool 0.4
- 位置衰减因子：0.85
- 融合策略：`blendedScore = heuristicScore * 0.7 + semanticScore * 0.3`

**ContextCompressor**（`context-compressor.ts`）：
- Auto-Compact：80% 阈值 + 30 秒冷却
- 主题分段 + 重要性评分 + 关键信息保留
- 动态上下文窗口：simple=1024 / medium=4096 / complex=8192

索引结构：邻接表（`outgoingIndex` / `incomingIndex`）

### 构建机制

- **实体添加** `addEntity()` (L236)：同名同类型自动合并属性
- **关系添加** `addRelation()` (L310)：验证实体存在性
- **文本抽取** `extractFromText()` (L1008)：正则 NER + 关系抽取，识别编程语言、框架、工具、文件路径、概念实体

### 查询机制

- **基础查询** `query()` (L374)：BFS 遍历
- **最短路径** `findPath()` (L466)：**Dijkstra 算法**，边权重 = `1 - rel.weight + 0.1`
- **多跳推理** `multiHopInference()` (L1341)：DFS 多跳推理

### 推理引擎

`infer()` (L645) 实现 4 种默认推理规则：
1. `is_a` 传递性（A is_a B, B is_a C → A is_a C）
2. `part_of` 传递性
3. `depends_on` 传递性
4. `is_a` 继承 `enables`

### 混合召回

- `hybridRecall()` (L1103)：图谱遍历 + 向量检索混合
- `hybridRecallAsync()` (L1226)：使用真实语义嵌入（OpenAI 1536 维 / TF-IDF 512 维）

### 工具暴露

4 个 LLM 工具：`kg_search`、`kg_find_path`、`kg_extract`、`kg_stats`

## 学习系统

### 1. SelfLearningSystem（交互学习）

`src/core/self-learning-system.ts`

**4 个核心数据结构**：

| 数据结构 | 说明 | 持久化文件 |
|---------|------|-----------|
| `records` | 学习记录（交互/错误/最佳实践） | `records.json` |
| `skills` | 技能等级（1-10） | `skills.json` |
| `knowledge` | 知识条目（带置信度） | `knowledge.json` |
| `patterns` | 交互模式（重复检测） | `patterns.json` |

**核心函数**：
- `learnFromInteraction(input, response, feedback?)` (L172)
- `learnFromError(error, context, recoveryAction?)` (L359)
- `learnBestPractice(practice, category, source)` (L414)
- `getLearningContext(query)` (L563)

**持久化边界保护**：
- `loadData()` (L994)：`record.content` 必须为 string，`tags` 必须为 array
- `saveData()` (L1047)：节流保存（5s），知识衰减（7 天降 0.1），低置信度清理（<0.15 且 30 天未访问），知识库上限 500 条
- 构造器必须 `fs.mkdirSync(dataPath, {recursive:true})`

### 3. MetaLearningSystem（元学习）

`src/core/meta-learning.ts` — "学会学习"能力

**6 种学习策略**：`incremental / batch / transfer / curriculum / active / reinforcement`

**核心函数**：
- `startSession(taskId, strategy?)` (L199)：推荐最佳策略和超参数
- `adaptHyperparameters()` (L362)：自适应调整（停滞→增大学习率，震荡→减小）
- `transferKnowledge(sourceTaskId, targetTaskId, knowledge)` (L397)：跨任务知识迁移
- `recommendStrategy(taskType)` (L301)：基于历史经验推荐

默认超参数：`learningRate=0.01, batchSize=32, explorationRate=0.1`

### 4. ReinforcementLearningSystem（强化学习）

`src/core/reinforcement-learning.ts` — Q-learning + 经验回放

**核心函数**：
- `chooseAction(state, availableActionIds?)` (L176)：ε-greedy 策略
- `learn(state, action, reward, nextState, done)` (L210)：存储经验 + Q 更新 + 回放 + ε 衰减
- `experienceReplay(batchSize)` (L264)：环形缓冲区去重采样
- `shapeReward(baseReward, factors)` (L325)：奖励塑造

**Q-learning 更新**：`Q(s,a) ← Q(s,a) + α[r + γ·max Q(s',a') - Q(s,a)]`

关键参数：α=0.1, γ=0.95, ε 从 1.0 衰减至 0.01（衰减率 0.995），环形缓冲区 10000 容量

### 5. ContinuousEvolutionSystem（持续进化）

`src/core/continuous-evolution-system.ts` — 每日自动竞品分析

**调度周期**：
- 每日 `runDailyCycle()` (L595)：爬取 → 分析 → 对比 → 路线图 → 学习注入
- 每周 `runWeeklyReview()` (L682)：综合评审
- 每月 `runMonthlyAssessment()` (L747)：战略重评估

**6 维对比框架**：`nlu_quality / reasoning_ability / tool_utilization / learning_efficiency / response_speed / user_experience`

**种子竞品**（15 个）：Claude Code / Codex / Cursor / Devin / Manus / Aider / Cline / GitHub Copilot 等

### 6. LearningEvalSystem（评估闭环）

`src/core/learning-eval-system.ts` — 5 个评估维度

`accuracy / efficiency / coverage / retention / adaptation`

**核心函数**：
- `recordSnapshot(dimensions, source, sampleSize)` (L102)
- `getAccuracy()` (L121)：最近 50 个快照平均
- `isAccuracyTargetMet()` (L131)：90% 准确率目标检测
- `createABTest(config)` (L135)：A/B 测试
- `completeABTest(testId)` (L158)：effect size + confidence 计算

## 进化指标

### `src/core/evolution-metrics.ts`

**5 大类别 17 个指标**：

| 类别 | 权重 | 指标 |
|------|------|------|
| intelligence | 0.30 | plan_quality / reflection_depth / strategy_adaptation / decision_accuracy |
| evolution | 0.25 | learning_velocity / knowledge_growth / experience_replay_efficiency / error_avoidance_rate |
| functionality | 0.15 | tool_coverage / feature_availability |
| performance | 0.15 | response_latency / token_efficiency / task_completion_efficiency |
| reliability | 0.15 | task_completion_rate / error_rate / doom_loop_rate |

### 两种指标来源

**suite 源**（source='suite'）：7 个套件覆盖 21 个指标
- thinking / memory / self_repair / computer_ops / code / inference / cross_platform
- 硬约束：`caseId` 必须与 `dimensions.ts` metric `id` 完全匹配

**new 源**（source='new'，运行时指标）：5 个
- `on_time_completion_rate`（delta）
- `quality_gate_pass_rate`（delta）
- `gap_probing_rate`（delta）
- `improvement_velocity`（direct）
- `regression_rate`（direct）

### read-and-reset 机制

```typescript
recordRuntimeValue(metricId, value, mode): void  // delta 累加 / direct 推入
flushRuntimeDeltas(): void                        // 读取 delta → 推入 50 样本滚动窗口 → 重置
getRuntimeAverage(metricId): number               // 滚动平均
getRuntimeP95(metricId): number                   // P95 百分位
```

硬约束：`source='new'` 指标必须用 read-and-reset delta 计数器维护 50 样本滚动平均。

## 工程约束

1. **原子写入**：所有 JSON 写入必须使用 `atomicWriteJson` / `atomicWriteJsonSync`
2. **SelfLearningSystem 构造器必须 mkdir**：`loadData()` 只读不创建目录
3. **运行时指标 read-and-reset**：`source='new'` 指标必须用 delta 计数器
4. **caseId 匹配契约**：能力评估套件 `caseId` 必须与 `dimensions.ts` metric `id` 完全匹配
5. **环形缓冲区优化**：`ReinforcementLearningSystem` 使用环形缓冲区替代 `array.shift()`
6. **懒加载设计**：`ContinuousEvolutionSystem` 构造器不执行 I/O
7. **确定性评分**：替换 `Math.random` 为基于能力的确定性评分
