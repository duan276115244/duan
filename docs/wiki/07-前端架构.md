# 07 前端架构

> 段先生前端基于 React 19 + Vite 6 + Zustand 5 + Tailwind CSS 4 构建，是一套面向"AI Agent 控制台"的单页应用。前端以对话区为核心，通过 SSE 流式接收 Agent 输出，同时以 `window.electronAPI` 桥接 Electron 桌面端能力。

**`handleSend` 主流程**（L830-L1065）：
1. Slash 命令拦截
2. 计划模式前缀注入（`/plan` 触发）
3. `@file` 引用上下文注入（完整文件内容进 message，仅摘要进 store）
4. 创建/获取对话，追加用户消息
5. AUTO 智能路由：若 `selectedModel === 'auto'` 且有 `electronAPI.agent.autoRoute`，先 IPC 获取最优模型
6. 调 `sendMessage(msg, history, onEvent, onDone, model, attachments)`
7. `onEvent` 回调按 `event.type` 分发：`think/text/tool_call/tool_result/error/warning/compact/plan`
8. `onDone` 回调：组装累积的 `accText/accThink/accTools` 为 assistant 消息写入 store，记录模型性能

**工具面板联动**（L984-L992）：`tool_call` 事件根据工具名自动激活对应面板 — `file_write/file_read` → editor、`shell_execute` → terminal、`browser_operate/desktop_open` → browser。

### 7.3 useChatStream SSE 流式处理详解（L325-L579）

**签名**：`sendMessage(message, history, onEvent, onDone, model?, attachments?)`

**Electron 模式**（L339-L455）：
1. 设置 120s 超时
2. 订阅 `electronAPI.agent.onStream(callback)`，callback 接收 `{ type, data/content }`
3. `data.type === 'stdout'`：Agent 子进程 stdout 输出，按行解析 SSE 格式
   - `data: [DONE]` → `onEvent({ type: 'done' })` + `onDone()`
   - `data: {json}` → `JSON.parse` 后 `onEvent(event)`
4. `data.type === 'done'` / `'error'` → 触发对应回调
5. 调 `electronAPI.agent.chat(message, history, model, attachments)` 发起对话
6. `doneCalled` 标志位防止 `onDone` 重复调用

**Web 模式**（L457-L548）：
1. `AbortController` + 120s 超时
2. `fetch('/api/chat/stream', { method: 'POST', body: ... })`
3. 401 → 重定向 `/config`
4. 收到响应后清除超时（流式可能持续很久）
5. `response.body.getReader()` + `TextDecoder` 按行解析 `data: ` 前缀

**事件类型**（`ChatEvent` 定义在 `types/index.ts` L100-L108）：
```typescript
type: 'text' | 'think' | 'tool_call' | 'tool_result' | 'agent_switch' | 'done' | 'error' | 'warning' | 'compact' | 'plan'
```

**`abort`**（L550-L564）：双模式中止 — Web 调 `abortRef.current.abort()`，Electron 调 `electronAPI.agent.onStop()`，并清理流监听 + 超时。

**卸载清理**（L567-L576）：组件卸载时中止请求 + 取消流监听 + 清除超时，防内存泄漏。

## 10. 构建配置 [vite.config.ts](file:///d:/good/jws/frontend/vite.config.ts)

```typescript
export default defineConfig({
  base: './',                                    // L7: 相对路径，支持 Electron file:// 加载
  plugins: [react(), tailwindcss()],             // L8-L11
  resolve: {
    alias: { '@': path.resolve(__dirname, './src') },  // L13-L15: @ 路径别名
  },
  build: {
    rollupOptions: {
      output: {
        manualChunks: {                          // L21-L26: vendor 分包
          'react-vendor': ['react', 'react-dom'],
          'markdown': ['react-markdown'],
          'icons': ['lucide-react'],
        },
      },
    },
  },
  server: {
    port: 5173,                                  // L30: dev server 端口
    host: '0.0.0.0',                             // L31: 允许外部访问（局域网调试）
    proxy: {
      '/api': {
        target: 'http://localhost:3001',         // L34: 后端 API 地址
        changeOrigin: true,
        secure: false,
      },
    },
  },
})
```

要点：
- `base: './'`：相对路径输出，让 Electron 能用 `file://` 协议加载打包后的 `index.html`
- **构建输出**：默认输出到 `frontend/dist`
- **vendor 分包**：3 个独立 chunk — `react-vendor`、`markdown`、`icons`，并行加载 + 长期缓存
- **路径别名**：`@` → `src`
- **dev proxy**：`/api/*` 转发到 `http://localhost:3001`（后端服务器），前端代码可直接 `fetch('/api/xxx')` 无 CORS 问题
- **构建命令**：`tsc -b && vite build` — 先类型检查再打包，类型错误会阻断构建

## 关键架构总结

1. **单一 store 架构**：全项目仅 `chatStore.ts` 一个 Zustand store，承载对话/系统状态/配置/主题/搜索等全部前端状态，用 `persist` 中间件持久化到 localStorage，`partialize` 只持久化 `conversations + currentConversationId`。

2. **SSE 双模式 + rAF 批处理**：`useChatStream` 实现 Electron IPC（`agent.onStream`）与 Web fetch SSE 双路径；`ChatArea.tsx` 用 `requestAnimationFrame` 把高频 chunk（20-50/秒）合并到每帧一次 setState，避免 ReactMarkdown 全量重新解析的性能瓶颈。

3. **对话页保状态策略**：用 `display: none` 而非条件渲染，避免切换页面时卸载 ChatArea 丢失流式状态。

4. **MarkdownRenderer 非独立组件**：Markdown 渲染逻辑内联在 `ChatArea.tsx`，共享模块级 `MARKDOWN_COMPONENTS` 常量。

5. **WorkflowBuilderPage 接线断裂**（严重 bug）：违反"四步全接线"契约，点击侧边栏"工作流"会导致主区域空白。PairingPage 完全孤立未接线。
