# 01 - 项目概览

## 项目定位

段先生 (Duan Xiansheng) v19.0 是一个**自我进化的自主 AI Agent 系统**，定位为"世界级 AI 助手"，对标 J.A.R.V.I.S.。项目名称来源于开发者代号，核心设计理念是"经验学习 + 本地推理 + 最优路径"。

### 核心能力

| 能力域 | 说明 |
|--------|------|
| 统一主循环 | Plan → Execute → Reflect → Learn 四阶段闭环 |
| 三端运行 | CLI（TUI 交互式）、Web（HTTP/SSE）、Desktop（Electron）三端互通 |
| 工具系统 | 140+ 内置工具，智能选择 + 熔断保护 + 沙箱隔离 |
| 记忆系统 | 分层记忆（L0/L1/L2）+ FTS5 + 向量检索 + 知识图谱 |
| 学习系统 | 6 大子系统：交互学习 / 元学习 / 强化学习 / 持续进化 / 评估闭环 |
| 自我进化 | Gödel Agent 架构 + 代码级 git 快照进化 + 自愈 + 竞品对比 |
| Agent 协作 | SubAgent / 团队 / 三 Agent 闭环 / Handoff / 后台 Agent |
| 意识模拟 | 5 种意识状态 + 双神经网络 + 7 种推理模式 |
| 多供应商 | 60+ 内置模型，AES-256-GCM 加密，三端配置同步 |

## 三端架构

段先生支持三种运行模式，共享同一套核心装配逻辑：

```
┌─────────────────────────────────────────────────────────────┐
│                    启动入口分发层                            │
├─────────────────┬─────────────────┬─────────────────────────┤
│  CLI 模式       │  Web 模式       │  Desktop 模式           │
│  npm run duan   │  npm run duan:web│  npm run duan:desktop  │
│      ↓          │      ↓          │      ↓                  │
│  src/entry.ts   │  src/entry.ts   │  desktop/main.js        │
│  (默认路径)     │  (web 子命令)   │  (Electron 主进程)      │
│      ↓          │      ↓          │      ↓ spawn            │
│  duan-v19.0.ts  │  web-server.ts  │  src/entry.js (子进程)  │
│  (TUI REPL)     │  (Express HTTP) │  → web-server.ts        │
└─────────────────┴─────────────────┴─────────────────────────┘
                        ↓ 共享
          ┌──────────────────────────────┐
          │  src/core/bootstrap.ts       │
          │  setupAgentLoop()            │
          │  ├─ createCoreModules()      │
          │  │   (150+ 模块实例化)       │
          │  ├─ createToolRegistry()     │
          │  └─ createAgentLoop()        │
          │      (EnhancedAgentLoop)     │
          └──────────────────────────────┘
                        ↓
          ┌──────────────────────────────┐
          │  enhanced-agent-loop.ts      │
          │  loop.run() 异步生成器       │
          │  Plan → Execute → Reflect    │
          │  → Learn                     │
          └──────────────────────────────┘
```

### 三端特点

| 模式 | 入口 | 交互方式 | 审批机制 |
|------|------|----------|----------|
| CLI | `entry.ts` → `duan-v19.0.ts` | TUI 交互式 REPL | inquirer 交互式审批 |
| Web | `entry.ts web` → `web-server.ts` | HTTP REST + SSE 流式 | HTTP API 异步审批 |
| Desktop | `desktop/main.js` → 子进程 | Electron IPC + SSE | IPC 弹窗审批 |

### 配置同步

三端共享 `~/.duan/config.json` 配置文件（v2.0 格式，AES-256-GCM 加密），通过 `fs.watch` 实时同步：
- CLI 修改配置 → 写入文件 → Web/Desktop 监听到变更自动重载
- 供应商热切换：`setActiveProfile()` → emit `config.external.changed` → ModelLibrary 重载 apiKey

## 设计理念

1. **统一装配**：`bootstrap.ts` 是唯一装配点，所有入口共享同一套模块实例化逻辑，避免双实例问题
2. **事件驱动**：`EventBus` 单例贯穿全项目，模块间解耦通信
3. **渐进式韧性**：三层熔断（ScalableToolRegistry 内置 → 独立 CircuitBreaker → 步骤级熔断）
4. **安全优先**：五层纵深防御（认证加密 → RBAC/ABAC → 沙箱 → MCP 安全 → 审计）
5. **自我进化**：从经验学习到代码级进化，git 快照保证可回滚
6. **类型安全**：全程 TypeScript，DIContainer 使用 `InjectionToken<T>` 泛型
