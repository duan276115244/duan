# 08 桌面应用

> 段先生桌面端基于 Electron 28 构建，采用**模块化工厂模式**：[main.js](file:///d:/good/jws/desktop/main.js) 作为入口协调器，将窗口管理、托盘、日志、配置、Agent 进程等子系统抽出到 `lib/` 子目录下的独立模块，通过 getter-callback 闭包模式共享状态。前端通过 `preload.js` 桥接的 `window.electronAPI` 调用主进程能力，主进程同时启动一个 Agent 后端子进程提供 AI 能力。

## 4. IPC 通信

**核心发现**：所有 IPC 都使用 `ipcMain.handle`（请求/响应模式），**没有 `ipcMain.on` 调用**。分为两种注册方式：
- 直接 `ipcMain.handle(channel, handler)` — 38 处（用于关键路径，无统一错误捕获需求）
- `safeHandle(channel, handler)` — 70 处（封装在 [safe-helpers.js](file:///d:/good/jws/desktop/lib/safe-helpers.js) L15-25，自动 try-catch 并返回 `{success:false, error}`，防 IPC 通道 reject 和错误信息泄露）

## 6. Webview 集成

desktop/ 目录下**没有 `<webview>` 标签的 HTML 代码**——webview 元素由前端 React 组件（[BrowserPanel.tsx](file:///d:/good/jws/frontend/src/components/BrowserPanel.tsx)）渲染。desktop/ 侧只负责**安全配置 webview 宿主环境**，集中在 [window-manager.js](file:///d:/good/jws/desktop/lib/window-manager.js)。

### 6.1 webviewTag 启用
主窗口、浏览器窗口、设置窗口的 `webPreferences.webviewTag: true`（L77/275/354）。`sandbox: false` 是必需的——以支持 partition 和 webviewTag。

### 6.2 `will-attach-webview` 安全加固（window-manager.js L95-102）

#### 证书错误处理（L131-160）—— **setCertificateVerifyProc 类型安全契约**
符合项目记忆中的"setCertificateVerifyProc 类型安全"契约：
```js
browserSession.setCertificateVerifyProc((request, callback) => {
  const { verificationResult, errorCode } = request;
  // 用 == 宽松比较防止 errorCode 类型差异（string '0' vs number 0）
  // String(verificationResult).trim() === 'net::OK' 防止空白字符差异
  if (!verificationResult || String(verificationResult).trim() === 'net::OK' || errorCode == 0) {
    callback(0); return;  // 信任
  }
  // 8 类可忽略证书错误（自签名 -201、过期 -200、CN 不匹配 -202 等）
  const ignorableErrors = [-200, -201, -202, -203, -204, -205, -207, -208];
  if (ignorableErrors.includes(errorCode)) {
    console.warn(...); callback(0); return;  // 允许加载但记录日志
  }
  console.error(...); callback(-3);  // 其他拒绝
});
```
**关键修复**（L134-137 注释）：原仅匹配 `verificationResult` 空字符串，导致 `net::OK`（truthy）被误判为错误，每会话 20 次证书验证失败。

### 9.2 `writeLogFile(level, args)`（L49-75）—— **webview 日志降噪三招**
符合项目记忆中的"webview 日志降噪三招"契约：

```js
// 第1招：GUEST_VIEW_MANAGER_CALL + ERR_ABORTED 完全静默
// （webview 导航被后续导航取消的正常现象）
if (/\bGUEST_VIEW_MANAGER_CALL\b.*ERR_ABORTED/.test(msg)) return;

// 第2招：electron: Failed to load URL + ERR_CONNECTION_RESET/TIMED_OUT 降级为 WARN
// （前端 did-fail-load 已处理并展示）
if (/electron: Failed to load URL.*ERR_CONNECTION_(RESET|TIMED_OUT)/.test(msg)) {
  if (level === 'ERROR') level = 'WARN';
}

// 第3招补充：ERR_ABORTED(-3) 单独出现也降级（about:blank 被后续 loadURL 取消）
if (level === 'ERROR' && /ERR_ABORTED \(-3\)/.test(msg)) level = 'WARN';
```

### 9.3 `setupFileLogging()`（L81-120）
- 创建 write stream（append 模式）
- **劫持 console.log/warn/error/info**：同时输出控制台和文件
- 捕获 `uncaughtException` 和 `unhandledRejection` 写 FATAL 级别
- 在 main.js L6491 启动（所有其他逻辑之前）

### 9.4 `closeFileLogging()`（L125-132）
在 `before-quit` 时调用（main.js L6609），确保所有日志落盘。

### 9.5 did-fail-load handler（window-manager.js L183-190）
```js
mw.webContents.on('did-fail-load', (_event, errorCode, errorDesc, validatedURL) => {
  if (errorCode === -3) return;  // 跳过 ERR_ABORTED
  console.error(`[主进程] 页面加载失败: ${errorCode} ${errorDesc} URL: ${validatedURL}`);
  if (IS_DEV) { mw.webContents.openDevTools({ mode: 'detach' }); }
});
```
这是"日志降噪三招"的第三招——`did-fail-load` handler 跳过 errorCode=-3。

## 关键架构总结

1. **模块化工厂模式**：main.js 是协调器，子系统全部抽到 `lib/`，通过 getter-callback 闭包共享状态（避免循环依赖和全局变量污染）。

2. **IPC 双轨注册**：`safeHandle`（70 处，自动 try-catch）用于常规业务，`ipcMain.handle`（38 处）用于关键路径；**全部为 invoke/handle 模式，无 on/send 单向通信**。

3. **安全契约全覆盖**：
   - CSP 注入（开发/生产差异化）
   - webview `will-attach-webview` 强制安全配置
   - `persist:browser` partition 权限白名单 + 证书宽松验证（类型安全比较）
   - `setWindowOpenHandler` 拦截新窗口重定向到 BrowserPanel
   - 浏览器窗口 `will-navigate` 协议白名单

4. **供应商切换两段式 IPC 契约**已验证（L3462-3507）：`config:unified:active:set` 只写配置 + 异步通知后端，`model:switch` 才是完整切换路径。

5. **webview 日志降噪三招**已验证（logger.js L64-69 + window-manager.js L185）：GUEST_VIEW_MANAGER_CALL 静默、CONNECTION_RESET/TIMED_OUT 降级、did-fail-load 跳过 -3。

6. **托盘常驻模式**：`window-all-closed` 空实现不退出，`close` 事件 preventDefault + hide，仅 `before-quit` 时真正清理。

7. **无 globalShortcut 注册**——快捷键功能在 desktop/ 层未实现。

8. **desktop/index.html 是死代码**（文件头已标 DEPRECATED D11），主进程通过 `getFrontendUrl()` 加载 `frontend/dist/index.html`。
